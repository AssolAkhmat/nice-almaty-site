# Nice Almaty бот — гайд по доработкам

Как устроен бот и как добавлять новые «штучки», не сломав то, что работает.
Все примеры — под вашу текущую кодовую базу.

---

## 1. Как это работает (одна картинка словами)

```
Веб-виджет (nice-almaty.html) ──► api/chat.js ─┐
                                                ├─► lib/bot.js  ask()
WhatsApp ──► Wazzup ──► api/wazzup-webhook.js ─┘        │
                                                        ├─ lib/sheets.js  getData()  ◄── Google Sheets (CSV)
                                                        │     └─ фолбэк ► data/houses/dom-*.md + data/availability.json
                                                        └─ lib/photos.js  (Drive-ссылки, маркеры фото)
                                                        │
                                                        ▼  DeepSeek (chat/completions)
```

**Три золотых правила этого кода — не нарушать:**
1. **Один «мозг».** Любая логика ответа живёт в `lib/bot.js`, а не в каналах. Каналы (`api/chat.js`, `api/wazzup-webhook.js`) — тонкие: принять сообщение → `bot.ask()` → отдать ответ.
2. **Никогда не падать.** `ask()` и `sheets.getData()` не бросают исключений — при любой ошибке отдают фолбэк. Всё новое делай так же.
3. **Приватность.** ФИО, телефоны, адреса, имена сотрудников **не должны попадать в промпт**. Новые поля из таблицы прогоняй через `scrubPII` (см. `lib/sheets.js`).

## 2. Карта файлов — куда лезть

| Хочешь изменить… | Файл |
|---|---|
| Поведение/тон/правила бота | `lib/bot.js` → `buildSystemPrompt()` |
| Данные из таблицы (колонки, вкладки) | `lib/sheets.js` |
| Очистку ответа (разметка, ссылки) | `lib/bot.js` → `cleanReply()` |
| Веб-эндпоинт / формат ответа | `api/chat.js` |
| Веб-виджет (вёрстка, рендер) | `nice-almaty.html` (`sendChat`, `addChatMsg`, `addChatImage`) |
| WhatsApp (приём/отправка) | `api/wazzup-webhook.js` |
| Запасные факты о домах | `data/houses/dom-*.md` |
| Тесты | `test/*.test.js` |

## 3. Рецепты (пошагово)

### Рецепт A — добавить поле к дому (напр. «Год постройки»)
1. **Таблица:** добавь колонку `Год` во вкладку `Дома`.
2. **`lib/sheets.js` → `mapHouses`:** прочитай поле:
   ```js
   year: field(o, ["Год", "Год постройки", "Year"]),
   ```
   Если поле текстовое и может содержать ПДн — оберни в `scrubPII(...)`.
3. **`lib/sheets.js` → `buildHouseText`:** выведи в блок дома:
   ```js
   if (h.year) lines.push(`- Год постройки: ${h.year}`);
   ```
4. Готово — модель увидит факт. Никаких изменений в `bot.js` не нужно.

### Рецепт B — добавить целую новую вкладку (как `Фото`)
Шаблон уже есть — смотри, как сделаны `Маршруты` и `Фото`:
1. `TABS.myTab = process.env.SHEETS_TAB_X || "МоёИмя"` (вверху `lib/sheets.js`).
2. Функция `mapMyTab(objs)` — маппинг строк в объекты (см. `mapRoutes`/`mapPhotos`).
3. В `getData()`: добавь вкладку в `Promise.all([... fetchCsvSafe(TABS.myTab)])`, распарси, положи в возвращаемый `data`.
4. Используй данные в `buildSystemPrompt` или в постобработке `ask()`.
Помни про фолбэк: `fetchCsvSafe` возвращает `null` при сбое и `""` при пустой вкладке — различай их, если пустота ≠ «удалить всё» (см. как сделано с `Маршруты`).

### Рецепт C — поменять правило/тон бота
Всё в `lib/bot.js` → `buildSystemPrompt()`, массив `lines`. Это обычный текст промпта.
Пример — запретить обсуждать конкурентов:
```js
lines.push("- Не сравнивай нас с конкретными конкурентами по названию; говори только о наших домах.");
```
После правок промпта **прогони бота вручную** (см. раздел 4) — маленькие формулировки сильно влияют.

### Рецепт D — новое «действие» через маркер (как `[ФОТО: Дом N]`)
Паттерн «маркер» — способ дать модели сделать действие, не давая ей выдумывать данные.
Пример: карта района `[КАРТА: Дом N]`.
1. В промпт (`buildSystemPrompt`): «если просят карту — добавь в конце строку `[КАРТА: Дом N]`».
2. Напиши чистую функцию-парсер по образцу `stripPhotoMarkers` в `lib/photos.js` (и **тест-оракул** в `test/`).
3. В `ask()`: после `cleanReply` распарси маркеры → преврати в данные (`attachments`/ссылки), вырежи маркер из текста.
4. Проведи данные в каналы (`api/chat.js` → JSON, `wazzup-webhook.js` → отправка).
Так модель никогда не выдаёт «сырой» URL/данные — только ссылается на дом, а код подставляет проверенное.

### Рецепт E — добавить/поправить язык
Языки: `lib/bot.js` → `LANG_NAME` и `FALLBACK`. В вебе язык приходит из `api/chat.js` (валидация `["ru","kz","en"]`). В WhatsApp язык не задаётся — модель зеркалит клиента. Чтобы добавить язык — расширь `LANG_NAME`, `FALLBACK`, список в `api/chat.js` и i18n-строки в `nice-almaty.html`.

### Рецепт F — защитить `/api/chat` от абьюза (rate-limit)
Сейчас у веб-эндпоинта нет лимита (в отличие от webhook с `?secret=`). Простой in-memory лимит по IP в `api/chat.js`:
```js
const HITS = new Map();
function limited(ip){
  const now = Date.now(), win = 60000, max = 20;
  const arr = (HITS.get(ip)||[]).filter(t => now - t < win);
  arr.push(now); HITS.set(ip, arr);
  return arr.length > max;
}
// в хендлере:
const ip = (req.headers["x-forwarded-for"]||"").split(",")[0] || "unknown";
if (limited(ip)) return res.status(429).json({ error: "too many requests" });
```
(На serverless память сбрасывается на холодный старт — для жёстких лимитов нужен Vercel KV/Upstash.)

### Рецепт G — сменить модель/провайдера
`lib/bot.js`: `MODEL`, `BASE_URL` (env `DEEPSEEK_MODEL`, `DEEPSEEK_BASE_URL`). Формат запроса — OpenAI-совместимый (`/chat/completions`). Другой OpenAI-совместимый провайдер меняется только env-переменными.

## 4. Как тестировать локально (без деплоя)

Бот ходит в сеть (Google Sheets + DeepSeek). В тестах подменяй `global.fetch`:

```js
// tест: мок и таблицы, и модели
process.env.SHEETS_ID = "TEST";
process.env.DEEPSEEK_API_KEY = "fake";
const CSV = { "Дома": "Дом,Район\n2,Парк Горького\n", "Маршруты": "Дом,ВУЗ,Автобусы,~минуты\n2,КБТУ,118,28\n",
              "Свободные места": "Дом,Пол,Свободно\nДом 2,муж,1\n", "Фото": "Дом,URL,Подпись\n" };
global.fetch = async (url) => {
  if (String(url).includes("deepseek"))
    return { ok: true, json: async () => ({ choices: [{ message: { content: "ответ модели [ФОТО: Дом 2]" } }] }) };
  const tab = decodeURIComponent(String(url).match(/sheet=([^&]+)/)[1]);
  return { ok: true, text: async () => CSV[tab] || "" };
};
const bot = require("./lib/bot.js");
bot.ask({ lang: "ru", message: "покажи фото", channel: "web" }).then(console.log);
```

**Чистые функции** (парсеры, нормализаторы) тестируй оракулом — см. `test/photos.test.js`.
Запуск: `node test/photos.test.js` (exit 0 = зелёно).
Синтаксис-проверка перед коммитом: `node -c lib/bot.js` (и т.д. по файлам).

## 5. Рабочий процесс (как делали эти изменения)

1. **Спроектируй контракт** (сигнатуры функций, поведение, примеры).
2. **Сначала напиши тест-оракул** (`test/x.test.js`) — он же ТЗ.
3. Реализуй (можно делегировать механику слабой/бесплатной модели через `opencode run -m opencode/big-pickle --auto`, дав ей спеку + тесты — она ограничена оракулом).
4. **Сам проверь** код и прогони тесты.
5. **Интеграцию в боевые файлы делай вручную** — там прячутся баги.
6. **Ревью независимой моделью:** `codex exec -s read-only -o REVIEW.md "..."`. Чини найденное, повторяй, пока не «YES на мерж».
7. **Не забудь `git add` новых файлов** — `require()` на незакоммиченный файл роняет прод.

## 6. Приватность — чек-лист на каждую доработку

- [ ] Новое текстовое поле из таблицы прогнано через `scrubPII`?
- [ ] Не выводим точный адрес (только район)? Правило в промпте на месте?
- [ ] URL/картинки только `http(s)` (нет `javascript:`/`data:`)?
- [ ] В веб-виджете вставка данных через `textContent`/DOM-свойства, а **не** `innerHTML`?
- [ ] ФИО жильцов/сотрудников не читаются и не уходят в модель?

## 7. Деплой-чек-лист

- [ ] Env на Vercel: `DEEPSEEK_API_KEY` (обяз.), `SHEETS_ID` (для таблицы), при WhatsApp — `WAZZUP_*`.
- [ ] Redeploy после изменения env.

- [ ] Черновики (`REVIEW_*.md`, `*_SPEC.md`) — **не** коммитить.
